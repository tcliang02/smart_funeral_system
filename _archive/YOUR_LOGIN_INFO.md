## üêõ **Troubleshooting**

### If you see "Access Denied":
- Make sure you logged in with **provider1** (not provider, not user1)
- Logout and login again

### If debug banner shows "Provider ID = null":
- Clear browser cache: Press Ctrl+Shift+Delete
- Run in console: `localStorage.clear()`
- Logout and login again

### If you get "Missing required fields" error:
- Check browser console (F12) for detailed logs
- Should show payload with provider_id = 3
- If provider_id is null, logout and login again

### If templates don't load:
- Check database: Run `SELECT COUNT(*) FROM addon_templates;`
- Should return 49
- If 0, re-run `buddhist_addon_system.sql`

### If customer can't see add-ons:
- Make sure you added services as provider first
- Check services are ENABLED (not disabled)
- Make sure customer is viewing provider1's package
